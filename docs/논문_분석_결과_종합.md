### 1.5 전처리 파이프라인 (스켈레톤)

```
1단계: 제외 선수 5명 삭제 (PARK Yewoon, TORRES QUEVEDO Ana 등)
2단계: TAKAHASHI Hiroatsu 국적 KOR → JPN 통일
3단계: 정상완주(is_normal=True) 필터링
4단계: 신체 데이터 병합 (athletes.csv → height_cm, weight_kg)
5단계: 숫자 강제 변환 + 핵심 결측치 제거
6단계: 파생변수 생성
        - Air_Density = (pressure_hpa × 100) / (287.05 × (air_temp + 273.15))
        - BMI = weight_kg / (height_cm / 100)²
7단계: finish < 50초 제거 (주니어스타트 이상치)
8단계: start_time < 4.5초 제거
9단계: 선수별 start_time 최고기록 + 0.6초 초과 제거
10단계: 선수별 speed 평균 - 5km/h 미만 제거
11단계: finish 상한선 적용 (정장환/송영민 54.5초)
12단계: 유지 선수 23명 외 athlete_id → NULL 처리
```

## 2. 스켈레톤 모델 성능

### 2.1 최종 모델 (XGBoost)

| 지표 | Train/Test Split (8:2) | 시계열 교차검증 (5-Fold) |
|------|----------------------|----------------------|
| **R²** | **87.86%** | **61.64%** |
| **RMSE** | 0.4622초 | ~0.80초 |
| 최적 파라미터 | max_depth=3, lr=0.1, n_estimators=300, subsample=0.8 | 동일 |

### 2.2 두 가지 R²의 의미

- **Train/Test R² 87.86%**: 전체 데이터에서 무작위로 20% 테스트셋을 분리했을 때의 설명력. 같은 시기의 데이터가 학습/테스트에 모두 포함될 수 있으므로 낙관적 추정치.
- **시계열 CV R² 61.64%**: 과거 데이터로만 학습하고 미래 데이터를 예측하는 현실적 시나리오. 시간 순서를 엄격히 지킨 정직한 추정치.
- **논문에서의 해석**: "환경 변수와 신체 변수만으로 미래 경기 기록의 약 62%를 설명할 수 있으며, 선수 개인 특성까지 포함하면 88%까지 설명 가능하다."

## 4. 모델 비교

### 4.1 시계열 교차검증 기준 (5-Fold)

| 순위 | 모델 | R² 평균 | R² std | RMSE 평균 | RMSE std |
|------|------|---------|--------|-----------|----------|
| 1 | MLR (선형회귀) | 61.84% | ±15.83% | 0.7487초 | ±0.0561 |
| 2 | XGBoost | 58.29% | ±11.75% | 0.7955초 | ±0.0292 |
| 3 | SVR | 56.97% | ±13.92% | 0.8039초 | ±0.0403 |
| 4 | Random Forest | 54.89% | ±13.22% | 0.8234초 | ±0.0222 |

> ※ 위 결과는 temp_avg 1개 사용 시점의 비교. 5구간 온도 적용 시 XGBoost가 61.64%로 상승.

### 4.2 해석

- 시계열 CV에서 MLR이 XGBoost를 근소하게 이기는 것은, **선형적 관계가 강한 데이터**(특히 start_time → finish)에서 MLR이 과적합 없이 안정적으로 작동하기 때문.
- XGBoost는 **비선형 패턴**을 잡아내는 데 강점이 있으며, Train/Test split에서는 87.86%로 압도적.
- **논문 주장**: "기본적 선형 관계는 MLR로도 포착 가능하나, 얼음 온도-마찰력의 비선형적 상호작용을 포착하기 위해 XGBoost가 필수적이며, 충분한 데이터가 확보될수록 XGBoost의 우위가 명확해진다."

## 6. 변수 그룹별 한계 기여도 (Tier 분석)

### 6.1 점진적 변수 추가 실험

변수 그룹을 순차적으로 추가하여 각 그룹의 **한계 기여도(Marginal Contribution)**를 측정.

| Tier | 변수 구성 | 변수 수 | 시계열 CV R² | 한계 기여 |
|------|---------|---------|-------------|---------|
| 1 | 스타트 시간만 | 1개 | 40.4% | 기준선 |
| 2 | + 환경 변수 (기온, 공기밀도, 이슬점, 얼음온도) | 5개 | 47.2% | **+6.8%p** |
| 3 | + 신체 변수 (신장, 체중, BMI) | 8개 | 56.8% | **+9.6%p** |
| 4 | + 선수 개인식별자 (athlete_id OHE) | 30개 | 58.3% | +1.5%p |

### 6.2 핵심 해석

- **스타트 시간** 하나만으로 기록 분산의 **40%**를 설명
- **환경 변수** 추가 시 **+6.8%p** 증가 → 기온·빙면온도·공기밀도가 유의미한 예측력 보유
- **신체 변수** 추가 시 **+9.6%p** 증가 → 가장 큰 한계 기여. 선수의 체형이 중력·공기저항에 미치는 영향
- **선수 ID** 추가 시 **+1.5%p**만 증가 → 선수 개인의 고유 주행 스타일 효과는 상대적으로 적음

### 6.3 논문 주장

> "측정 가능한 물리적 변수(스타트 기록, 기상 환경, 신체 특성)만으로 주행 기록의 57%를 설명할 수 있으며, 이는 지도자의 주관적 경험에 의존하던 기존 코칭 방식을 데이터 기반 의사결정으로 전환할 수 있는 충분한 근거를 제공한다."

## 7. 변수 제거 실험 (Ablation Study)

### 7.1 결과 (8개 핵심 변수 기준, 시계열 CV)

기준선 R²: 56.94% (8개 변수 전체)

| 제거 변수 | 제거 후 R² | 변화량 | 해석 |
|----------|----------|--------|------|
| **start_time** | 24.81% | **-32.13%p** | 제거 시 성능 붕괴. 압도적 핵심 변수 |
| **temp_avg (얼음온도)** | 54.20% | **-2.74%p** | 환경 변수 중 가장 중요 |
| BMI | 54.92% | -2.02%p | 신체 변수 중 가장 중요 |
| Air_Density (공기밀도) | 55.42% | -1.52%p | 공기저항 관련 기여 |
| weight_kg | 55.55% | -1.38%p | 중력 효과 |
| height_cm | 56.82% | -0.12%p | 제거해도 거의 영향 없음 |
| dewpoint_c (이슬점) | 57.84% | +0.90%p | 제거 시 오히려 소폭 개선 |
| air_temp (기온) | 58.43% | +1.49%p | Air_Density에 이미 반영 → 중복 제거 효과 |

### 7.2 핵심 발견

1. **스타트 시간**: 제거 시 R²가 32%p 하락하여 모델이 사실상 무력화됨. "스타트가 가장 중요하다"는 현장의 경험적 지식을 정량적으로 입증.
2. **얼음 온도 (temp_avg)**: 환경 변수 중 유일하게 제거 시 유의미한 성능 하락(-2.74%p). 프로포졀의 핵심 주장인 "빙면 온도가 기록에 치명적 영향"을 데이터로 증명.
3. **기온·이슬점**: 개별적으로는 Air_Density 파생변수에 정보가 흡수되어 중복. 독립적 기여는 미미.

## 8. 구간별 빙면 온도 심화 분석

### 8.1 배경

프로포졀에서 "44개 센서의 구간별 얼음 온도 활용"을 핵심 차별점으로 제시하였으나, 기존 모델에서는 전체 평균(temp_avg) 1개 값만 사용.

### 8.2 데이터 구조

평창 알펜시아 슬라이딩 센터의 42개 트랙 구간에 설치된 90개 적외선 온도 센서에서 수집한 빙면 온도를 5개 주행 구간으로 매핑:

| 주행 구간 | 트랙 센서 | 변수명 | 지대 |
|----------|---------|--------|------|
| Start → Int.1 (커브4) | Seg 1~9 | ice_zone1 | 고지대 상단 |
| Int.1 → Int.2 (커브7) | Seg 10~16 | ice_zone2 | 고지대 중단 |
| Int.2 → Int.3 (커브12) | Seg 17~28 | ice_zone3 | 고지대 하단~중지대 |
| Int.3 → Int.4 (커브15) | Seg 29~34 | ice_zone4 | 중지대~저지대 |
| Int.4 → Finish | Seg 35~42 | ice_zone5 | 저지대 |

### 8.3 구간별 온도 차이 사례 (2026-03-17)

| 구간 | 평균 빙면온도 |
|------|------------|
| ice_zone1 (Start→Int.1) | -6.29°C |
| ice_zone2 (Int.1→Int.2) | -6.02°C |
| ice_zone3 (Int.2→Int.3) | -6.15°C |
| ice_zone4 (Int.3→Int.4) | **-7.40°C** |
| ice_zone5 (Int.4→Finish) | -5.77°C |

> 같은 날, 같은 트랙에서도 구간별로 최대 **1.63°C 차이** 발생. 이는 일조량, 그늘, 냉각 설비 위치에 따른 미세기후 차이를 반영.

### 8.4 성능 비교

| 모델 | 온도 변수 | 데이터 건수 | Train/Test R² | 시계열 CV R² | 시계열 CV std |
|------|---------|-----------|--------------|------------|-------------|
| A: 기존 | temp_avg 1개 | 1,288건 | 81.61% | 54.97% | ±14.03% |
| B: 5구간만 | ice_zone1~5 | 832건 | 80.42% | 60.13% | ±8.58% |
| C: 전체 | temp_avg + 5구간 | 731건 | 82.12% | 54.87% | ±11.74% |
| **D: 5구간+fallback** | **ice_zone1~5 (없으면 temp_avg)** | **1,389건** | **81.91%** | **61.64%** | **±8.45%** |

### 8.5 핵심 결과

**Model D (5구간 온도 + fallback) 채택 시:**
- 시계열 CV R²: **54.97% → 61.64% (+6.67%p 상승)**
- 안정성(std): **±14.03% → ±8.45% (40% 개선)**
- 데이터 건수: 1,288건 → 1,389건 (오히려 증가)

### 8.6 구간별 SHAP 기여도

|------|------|------|------|

| ice_zone2 | Int.1→Int.2 (7번 커브 부근) | 0.0772 | 2위 |
| ice_zone1 | Start→Int.1 (스타트 구간) | 0.0609 | 3위 |

### 8.7 논문 주장

> "전체 평균 빙면 온도(temp_avg) 대신 주행 구간별로 세분화된 5개 빙면 온도 변수를 사용할 경우, 시계열 교차검증 R²가 6.67%p 상승하고 예측 안정성이 40% 개선되었다. 특히 Int.3~Int.4 구간(15번 커브 부근)의 빙면 온도가 최종 기록에 가장 큰 영향을 미치는 것으로 나타났으며, 이는 해당 구간이 고속 커브가 연속되어 얼음 마찰계수 변화에 가장 민감한 구간임을 시사한다. 이 결과는 44개 센서의 구간별 데이터 활용이 단순 평균 대비 유의미한 예측력 향상을 가져옴을 실증적으로 입증한다."

## 8.5. MLR 코칭 인사이트 (다중선형회귀 변수별 가중치)

### 목적

XGBoost는 높은 예측력을 제공하나 '블랙박스' 문제가 있다. 다중선형회귀(MLR)를 병행하여 각 변수가 피니시 기록에 미치는 영향을 **정량적 계수**로 도출함으로써, 지도자가 즉시 이해하고 활용할 수 있는 코칭 지표를 제공한다.

### MLR 성능

| 지표 | 값 |
|------|-----|
| R² | 70.57% |
| RMSE | 0.7194초 |

### 변수별 회귀 계수 (가중치)

| 변수 | 계수 | 해석 |
|------|------|------|
| **start_time** | **+3.445** | 스타트 1초 느리면 → 피니시 약 3.45초 악화 (0.1초당 ~0.35초) |
| Air_Density | +11.433 | 공기밀도 0.01 증가 → 피니시 약 0.11초 악화 |
| BMI | +0.194 | BMI 1 증가 → 피니시 약 0.19초 악화 (체형 영향) |
| height_cm | +0.075 | 키 1cm 증가 → 피니시 약 0.075초 악화 (공기저항 면적) |
| air_temp | +0.073 | 기온 1°C 상승 → 피니시 약 0.07초 악화 |
| dewpoint_c | +0.017 | 이슬점 1°C 상승 → 피니시 약 0.02초 악화 |
| weight_kg | **-0.070** | 체중 1kg 증가 → 피니시 약 0.07초 **단축** (중력 효과) |

### 코칭 활용 예시

> "현재 평창 트랙의 빙질·기상 환경(공기밀도 1.16kg/m³, 얼음온도 -7°C)에서, A 선수가 스타트 기록을 0.1초 단축하면 피니시 약 0.35초 단축이 기대된다. 또한 규정 허용 범위 내에서 체중을 2kg 증가시키면 피니시 약 0.14초 추가 단축이 가능하다."

이처럼 MLR은 XGBoost의 높은 예측력을 보완하여, 지도자에게 **"정확히 무엇을 얼마나 바꾸면 몇 초 빨라지는가"**라는 구체적 코칭 기준선(Baseline)을 제공한다.

- 파이프라인에서 `use_physical=False`로 설정된 이유

**원인 4: 절대적 데이터 부족**
- 전처리 후 316건 (스켈레톤 1,389건의 23%)
- GridSearchCV 3-fold CV 시 각 fold가 ~70건에 불과

**원인 5: 이상치 필터링 부재**
- 스켈레톤처럼 start_time 기준(+0.6초), speed 기준(-5km/h) 이상치 필터링 없음
- is_normal 태깅도 불완전 (빈값 다수)

### 12.3 연구방법 추가 사항

**시계열 교차검증 (반드시 추가)**
> 본 연구는 데이터 누수(Data Leakage)를 방지하기 위해 시간 순서를 엄격히 지킨 시계열 교차검증(TimeSeriesSplit, n_splits=5)을 적용하였다. 이는 과거 데이터만으로 학습하고 미래 데이터를 예측하는 현실적 시나리오를 재현한 것이다.

**SHAP 분석 (반드시 추가)**
> 인공지능의 '블랙박스' 문제를 해결하기 위해 SHAP(SHapley Additive exPlanations) 기법을 적용하여 각 변수의 기록에 대한 한계 기여도를 정량화하였다.

### 12.4 R² 보고 방식

프로포졀에서 "90%+ 예측 정확도"를 암시하는 부분 수정 필요:

> "본 모델은 Train/Test 분할 기준 R² 87.86%, 시계열 교차검증 기준 R² 61.64%를 달성하였다. 이는 측정 가능한 환경·신체 변수만으로 미래 경기 기록의 약 62%를 설명할 수 있으며, 동일 시기 내 예측 시 88%까지 설명 가능함을 의미한다."

### 12.5 AI 대시보드 범위 조정

- 프로포졀에서 "AI 대시보드 구축"을 주요 목표로 제시하였으나, 박사 논문의 핵심은 "모델 개발 및 검증"에 집중하는 것이 적절
- 대시보드는 이미 배포 중 (https://skeleton-analysis-production-d1bb.up.railway.app/)
- 논문에서는 "실용화 사례" 또는 "향후 연구"로 간략히 언급

### 12.6 봅슬레이/루지 처리

삭제가 아니라 **"모델 일반화 검증"** 장으로 포함:

> "스켈레톤에서 개발된 예측 파이프라인을 봅슬레이·루지에 동일하게 적용한 결과, 종목별 구조적 차이(종목 혼재, 신체 데이터 결측 등)로 인해 예측력이 유의미하게 낮아졌다. 이는 환경 변수 기반 예측 모델의 유효 조건을 규명하는 중요한 발견이다."

### 14.3 대시보드 예측 사례

**입력**: 여찬혁(ATH-81CCC6D3), 스타트 4.952초
**기상**: 기온 4.1°C, 습도 71.6%, 기압 926.3hPa
**빙면온도**: -7°C (수동 입력)

| 모델 | 예측 피니시 | R² |
|------|-----------|-----|
| Poly+Ridge | 53.19초 | 45.6% |
| XGBoost | 53.34초 | 79.6% |
| **앙상블** | **53.31초** | - |

**자동 생성 코칭 TIP:**
- 스타트 0.1초 단축 시 피니시 약 0.25초 단축. 현재 목표: 4.952초
- 이슬점(-0.6°C) > 빙면(-7°C) → 서리로 마찰 증가 가능 (+0.1~0.3초)
- Int.4(15번 커브)가 피니시 예측의 52.3%를 결정. Turn 13 진입 속도 최적화 권장
